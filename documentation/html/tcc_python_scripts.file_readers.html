assign_diameters(uij_range, diameter)[source]¶

Assign diameters to the particles across the given ID range.

Diameters will only be assigned if the range describes a single-species interaction. Any other range is potentially a nonadditive interaction so an exception is raised as a warning.

Parameters:
  • uij_range – xml entry describing the range of the interaction
  • diameter – length scale of these hard spheres
Raises:

NonadditiveError – if the given range is a cross-species interaction, so potentially the dynamo file describes a nonadditive potential.
static is_hard_sphere(interaction)[source]¶

Determine whether an interaction encoded in the dynamo XML tree is a hard sphere interaction or not.

There are two ways to encode hard spheres: either explicitly as a hard sphere, or as a square well with zero well depth (although the latter should be avoided as it results in a less efficient simulation). This helper function tests for both eventualities.

Parameters:interaction – xml entry for the interaction
Returns:boolean stating whether the interaction is a hard sphere

verify_cross_species_interactions(uij_range, diameter, eps=1e-12)[source]¶

Check whether a cross-species hard sphere interaction is consistent with the known diameters of each species, assuming additive interactions.

Additive cross-species interactions have size sigma = 0.5*(sigma_i + sigma_j) where sigma_{i,j} are the diameters of the individual species. If the diameter does not match this expectation then the interaction is not additive.

Parameters:
  • uij_range – xml entry describing the range of the interaction
  • diameter – length scale of this cross-species hard sphere interaction
  • eps – machine tolerance for the test of additivity
Raises:

NonadditiveError – if the interaction is not additive
